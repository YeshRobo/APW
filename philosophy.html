Systems Philosophy

Agents Are Systems That Change Systems

AI agents are one modern form of a broader engineering pattern: systems that perceive, process, act, and transform state.

View Portfolio View Agent Loop

Agency is a loop, not a label.

AI agents are not the only agents.

When people say "agent" today, they often mean an LLM-powered assistant, chatbot, or software automation. I use the word more broadly. An agent does not have to be a chatbot, and it does not have to be human-like.

An agent only needs to close a loop between perception, internal processing, action, and environmental change. By that definition, many engineered and natural systems can be reimagined as agents.

My Definition

An agent is a system that senses or receives information about a current state, processes that information through an internal core, and acts to produce a changed state.

Perception

The system receives signals, measurements, observations, or context from the world.

Agent Core

The system compares, reasons, plans, predicts, retrieves, controls, or decides what should happen next.

Action

The system produces an output through code, force, motion, communication, or structured behavior, bounded by what the action channel can safely do.

State Change

The system leaves the environment, task, process, or user in a different condition than before.

The Simplest Agent: A Thermostat

A thermostat is one of the simplest examples of this loop. It does not reason like a human or plan like an AI system, but it still senses, compares, acts, and changes its environment.

Current System State

The room is colder or warmer than the desired temperature.

Perception

A temperature sensor reads the current room temperature.

Agent Core

The thermostat compares the measured temperature against the target setpoint.

Action

It turns heating or cooling on or off.

Changed System State

The room temperature moves toward the target.

This is not intelligence in the human sense. It is agency at a basic control level: perception, internal comparison, action, and then perceiving the changed room again.

Current System State to Changed System State

The framework describes how an agent changes a system from one state to another.

Current System State Perception Agent Core Action Changed System State

From a thermostat to an AI agent, the loop stays recognizable. Feedback is not a separate magic layer; the changed system state becomes the next current system state when the agent perceives the environment again.

What changes between agents is the depth of perception, the complexity of the Agent Core, and the range of possible actions.

What Changes Between Agents

Agents differ by degree, not just by category. A thermostat and a human are not equally intelligent, but both can be studied through the same loop.

Perception Depth

How much of the environment the system can sense, read, or infer.

Memory and Context

How much history, state, or situational context the system can maintain.

Decision Complexity

Whether the core is a comparison, controller, policy, planner, reasoning model, or organization process.

Core Constraints

The goals, rules, safety policies, and operating limits that shape decisions inside the Agent Core.

Action Limits

The physical constraints, actuator limits, interfaces, and safety mechanisms built into how action is executed.

Re-Perception

After action changes the system, the agent perceives the environment again and continues the loop.

Spectrum of Agents

The point is not that every agent is equally intelligent. The point is that different systems can share the same structure at different levels of complexity.

Thermostat

A basic control loop compares temperature against a setpoint and switches heating or cooling.

Gimbal

A control algorithm converts IMU and encoder feedback into motor commands.

Robot

A learned policy selects steering and motion actions while respecting safety constraints.

AI Agent

An LLM, tools, memory, and retrieval pipeline convert user intent into software or information actions.

Human

The mind receives perception, reasons about the situation, and selects action.

Organization

People, rules, plans, communication, and decision structure coordinate action at larger scale.

What the Agent Core Can Be

The Agent Core is the decision-making center of the agent. It receives perception, forms an internal representation of the current system state, applies goals and constraints, and selects or produces the next action.

In engineering terms, the Agent Core can be a setpoint comparison, controller, policy, planner, retrieval pipeline, reasoning process, or organizational decision structure. The constraints inside it decide what actions are valid before the action is executed.

Why This Matters for Robotics, AI, and Systems Engineering

This framing makes engineering projects easier to compare. A robot, an embedded controller, an AI tool, a software analytics system, and a human-machine workflow can all be explained by what they perceive, how they process it, what action they take, and what system state changes.

This is the lens I use across my portfolio: build systems that observe the world, make structured decisions, act within constraints, and create measurable outcomes.
